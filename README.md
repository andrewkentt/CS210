This project involved creating a C++ program that reads a text file of grocery items and counts how many times each item appears. The program allows the user to search for a specific item, print all item frequencies, and display a histogram, and it automatically creates a backup file called frequency.dat when the program starts. I organized the program using a class with public and private sections to keep the code structured and readable. Using a map made it straightforward to store and update item frequencies efficiently. I also added input validation to prevent the program from crashing if the user entered an invalid menu choice.

The most challenging part of the project was getting the file paths configured correctly in Visual Studio, since the program initially could not locate the input file. After adjusting the working directory, the file input and backup creation worked as expected. If I were to improve the program, I would make it more flexible by allowing the user to choose a different input file and possibly improve the output formatting. This project strengthened my understanding of file input and output, maps, class structure, and debugging in C++. I focused on maintainability by using clear variable names, separating functionality within the class, and including comments to make the code easier to understand and modify.# CS210
